# 日本賽馬共同語言資料庫設計

日期：2026-06-11

## 目標

建立一套專為日本賽馬設計、以 Markdown 為主的共同語言資料庫。第一版聚焦日本賽馬術語，服務 Coding agent / Codex 在專案內查詢、翻譯、統一用語與理解語境。

資料庫應同時滿足：

- 可作為翻譯對照表。
- 可作為低雜訊賽馬知識庫。
- 可用 `rg` 或一般文字搜尋快速定位詞條。
- 可追溯來源可信度。
- 避免自創無根據譯名，尤其是馬名。

## 非目標

- 第一版不建立完整 API、網站或資料庫服務。
- 第一版不大量收錄知名馬名。
- 第一版不追求百科式長文解釋。
- 沒有可靠中文譯名的馬名，不主動翻譯。

## 建議目錄

```text
docs/

      basic-racing.md

      race-analysis.md
    references/
```

## 檔案責任

### `INDEX.md`

AI 查詢入口，只保留：

- 知識庫用途。
- 查詢流程。
- 分類地圖。
- 常用搜尋關鍵字。
- 各檔案責任邊界。

AI 應先讀 `INDEX.md`，再依分類讀取目標詞條檔，避免一次載入過多雜訊。

### `terms/basic-racing.md`

收錄基礎觀賽、投注、馬場、名次、距離、出走、讓磅等術語。

### `terms/japanese-racing-system.md`

收錄日本賽制與賽事分級術語，例如中央競馬、地方競馬、重賞、G1/G2/G3、Listed、OP、條件戰等。

### `terms/race-analysis.md`

收錄比賽分析與馬匹能力術語，例如腳質、末腳、折合、掛かる、馬場適性、調教、血統傾向等。

### `references/source-policy.md`

記錄來源分級、引用規則、來源 URL 與使用範圍。

### `references/glossary-style.md`

記錄譯名選擇規則，包括 HKJC 優先、暫定譯名、同義詞處理、馬名處理規則。

## 詞條格式

分類詞條檔以 Markdown 表格為主。欄位固定，內容保持低雜訊。

```markdown
| ID | 主譯名 | 日文 | 英文 | 分類 | 定義 | 使用語境 | 相關詞 | 譯名狀態 | 來源等級 |
|---|---|---|---|---|---|---|---|---|---|
| RA-001 | 末腳 | 末脚 | late kick / finishing speed | 比賽分析 | 馬匹在最後階段加速衝刺的能力。 | 評估直路衝刺與追込型馬匹時使用。 | 腳質、上がり | 常用 | B |
```

欄位定義：

- `ID`：固定代碼，格式依分類區分，例如 `BR-001`、`JRS-001`、`RA-001`。
- `主譯名`：AI 優先使用的正體中文詞。
- `日文`：原詞，保留漢字、假名或常見寫法。
- `英文`：有穩定英文對應詞時填寫；沒有時可留空。
- `分類`：控制語境，避免不同領域混用。
- `定義`：1 句為主，最多 2 句。
- `使用語境`：說明什麼情況使用此詞。
- `相關詞`：只列直接相關詞，不建立長篇知識網。
- `譯名狀態`：只能使用 `官方`、`常用`、`暫定`、`需確認`。
- `來源等級`：只能使用 `A`、`B`、`C`、`D`。

詳細來源不放進每一列，避免表格噪音過高。詞條只記來源等級；來源 URL 與引用策略集中放在 `references/source-policy.md`。

## 來源分級

```text
A = 高可信中文來源
    HKJC 官方中文資料、香港賽馬會賽例/投注/馬匹資料、JRA 官方繁中資料。

B = 高可信原文或英文來源
    JRA 官方日文/英文資料、Horse Racing in Japan、netkeiba 英文版、JBIS 等日本賽馬資料庫。

C = 背景補充來源
    維基百科、日英詞典、一般賽馬介紹文章。只能補背景，不單獨決定主譯名。

D = 待確認
    由語境推定、社群常用、暫無正式來源。可收錄，但必須標為暫定或需確認。
```

## 譯名決策規則

1. 有 HKJC 或 JRA 繁中譯名時，優先採用。
2. 沒有官方中文譯名，但有 JRA、Horse Racing in Japan、netkeiba 英文版等可信語境時，可建立正體中文暫定譯名。
3. 維基百科只用來補充背景或確認常識，不作為唯一權威來源。
4. 同一術語有多個中文說法時，只選一個 `主譯名`，其他寫入 `相關詞` 或註記。
5. `暫定`、`需確認` 詞條可以存在，但不得被視為權威譯名。
6. 馬名比術語更保守：沒有可靠中文譯名時，不主動翻譯。

## 馬名處理規則

名馬或馬名不是第一版核心。若未來新增馬名模組，建議放在：

```text
docs/
  racing-language/
    horses/
      notable-horses.md
```

馬名規則：

- 有 HKJC、JRA 繁中、維基中文或其他可信正體中文來源時，才記錄中文名。
- 沒有中文名時，保留日文名與英文或羅馬字。
- 不為了資料完整性自創馬名翻譯。
- 馬名模組可作為後續版本，不列入第一版必要交付。

## 維護流程

新增術語時：

1. 確認分類：基礎觀賽、日本賽制、比賽分析。
2. 搜尋是否已有近似詞，避免重複詞條。
3. 補齊核心欄位：ID、主譯名、日文、英文、定義、使用語境、譯名狀態、來源等級。
4. 若來源不足，標示 `D` 與 `需確認`，不得偽裝成正式譯名。
5. 日本特有制度詞放在 `japanese-racing-system.md`。
6. 分析用語放在 `race-analysis.md`。
7. 基礎觀賽與一般賽馬詞放在 `basic-racing.md`。

## 驗證方式

文件驗證以一致性與可查詢性為主：

- Markdown 表格欄位一致。
- 每個詞條都有唯一 ID。
- `譯名狀態` 只使用固定值。
- `來源等級` 只使用 `A`、`B`、`C`、`D`。
- `INDEX.md` 指向所有分類檔。
- 抽樣確認來源策略沒有違反「不硬翻馬名」原則。
- 用 `rg` 檢查 ID 重複、主譯名重複與分類一致性。

## 成功條件

- 第一版資料庫架構可讓 Codex 從 `INDEX.md` 判斷要查哪個分類檔。
- 詞條內容足夠翻譯與理解語境，但不包含不必要長文。
- 來源可信度清楚，不把暫定資料誤標為官方資料。
- 術語可逐步擴充，不需要改變整體結構。
- 馬名沒有可靠中文譯名時不硬翻。
